# Contributing

Thanks for your interest in improving PetMorph's public repository.

## How to contribute

1. Open an Issue first for major changes.
2. Keep proposals scoped to public docs, issue templates, and public references.
3. Use clear reproduction steps for bug reports.
4. Keep personal data, secrets, and private implementation details out of issues.

## What can be contributed here

- Documentation improvements
- Public roadmap suggestions
- Bug reports and UX feedback on the live product
- Clarity improvements to security and support docs

## What is out of scope

- Direct changes to private production source code
- Requests for internal credentials, infrastructure, or private datasets

## Pull requests

PRs that improve public docs and issue quality are welcome.
